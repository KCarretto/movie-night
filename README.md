# Plot Polls

Plot Polls is a serverless, peer-to-peer movie voting app built with React and
[Vite](https://vitejs.dev/). It is served as a static site from the `build/`
directory (the GitHub Pages root).

## What it does

- Create a room and invite friends with a join link or QR code
- Nominate movies and vote in ranked order
- Pick a winner from group results
- Save personal watch history and ratings locally
- Generate recommendations from shared taste vectors

## Tech stack

- React 18 + Vite (single-page app, output to `build/`)
- WebRTC networking via PeerJS
- Tailwind CSS from CDN
- QR generation/scanning via qrcode.js and html5-qrcode
- Protobuf catalog (`movies.pbf`) + binary embeddings (`embeddings_part*.bin`),
  served from `build/data/`

## Project layout

- `src/lib/` — framework-agnostic logic (catalogue, embeddings, recommendation
  engine, instant-runoff, vector math, runtime store)
- `src/state/` — PeerJS room controller + the React `useStore` hook
- `src/ui/`, `src/components/`, `src/screens/`, `src/modals/` — composed React UI
- `public/data/` — `movies.pbf` + `embeddings_part*.bin` (copied to `build/data/`)
- `index.html` — Vite entry (CDN deps, styles, mount point)

## Develop & build

```bash
npm install      # install dependencies
npm run dev      # local dev server with hot reloading
npm run build    # produce the static site in build/
npm run preview  # preview the production build locally
```

The production build is published to GitHub Pages by
`.github/workflows/deploy-pages.yml` on every push to `main`.

## Data pipeline

Movie catalog and embedding files are generated by scripts in `scripts/` and
updated by `.github/workflows/sync-movies.yml`, which writes them into
`public/data/`.

See `scripts/README.md` for details on regenerating `movies.pbf` and the
embedding binaries.
